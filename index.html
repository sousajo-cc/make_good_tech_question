How to ask a technical question

<Ask Technical Question>

   <Context>

      Please sum up what you already did. What you've tried. The command(s) that you used, the error message(s), and the full logs or a path to them.

   </Context>

   <Avoid Screenshots>

      While screenshots might help and give some context to the question, please remember one cannot copy text from them, and it is not searchable by default. Lots of waste!

   </Avoid Screenshots>

   <Readability>

      Use markdown. Makes the question/example more readable and easy to reason about.

   </Readability>

   <No environment assumptions>

      Post full commands. People are not familiar with your aliases, .bashrc, and if you do rely on some local environment assumptions please mention those and how should they be reproduced.

   </No environment assumptions>

   <Minimal reproducible example>

      If one manages to do it, it really helps to narrow down the source of the problem, and helps people to help you.

   </Minimal reproducible example>

   <Search>

     Your question might already been asked and answered. Take some time to search it up, before firing it up.

   </Search>

   <Right forum?>

      Pose that question to yourself. Asking for help on a C++ issue on movies.stackexchange might not be ideal.

   </Right forum?>

</Ask Technical Question>



What's the goal of this document?
Less waste and better communication. How can we better help each-other.

How can one contribute?
Open a pr or an issue.
