How to make a (better) technical question

  • Context Please sum up what you already did. What you tried. The command that you used, the error message, and path to the full logs.
  • No Screenshot only questions While screenshots might help and give some context to the question, please remember one cannot copy text from them.
  • Readability Use markdown. Makes the question/example more readable and easy to reason about.
  • No environment assumptions Post full commands. People are not familiar with your aliases and .bashrc, and if you do rely on some local environment assumptions please mention those and how should they be reproduced
  • Create a minimal reproducible example If one manages to do it, it really helps to narrow down the source of the problem.
  • Search for similar questions It might actually also work for you and solve your problem.
  • Is this the right forum? Pose that question to yourself. Asking for help on a C++ issue on movies.stackexchange might not be ideal.
